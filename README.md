# 🎭 מתחזה — Imposter (Multiplayer)

משחק מסיבה בסגנון ["מתחזה"](https://imposter-game.fun/), אבל **בלי להעביר טלפון אחד בין כולם**.
כמו ב-Kahoot: המנחה פותח משחק על מסך אחד, וכל שחקן מצטרף מהטלפון שלו דרך **קוד**, **לינק** או **QR**.

## איך משחקים

1. **המנחה** לוחץ *"פתח משחק חדש"*, בוחר הגדרות ומקבל מסך משותף עם קוד + QR.
2. **השחקנים** נכנסים לאתר, סורקים את ה-QR (או מקלידים את הקוד) ומזינים שם.
3. המנחה מתחיל את הסבב. כל שחקן רואה **במכשיר שלו** את התפקיד:
   - רוב השחקנים מקבלים את **המילה הסודית**.
   - שחקן אחד (או יותר) הוא **המתחזה** — לא מכיר את המילה (רק את הקטגוריה, אם הוגדר כך).
   - כרטיס התפקיד **מוסתר כברירת מחדל** ונראה זהה אצל כולם — לחיצה חושפת, לחיצה נוספת מסתירה. אפשר להניח את הטלפון על השולחן בבטחה.
4. בתורות, כל אחד אומר **מילה אחת** שקשורה למילה הסודית. המתחזה מנסה להשתלב.
   - המשחק מציג **של מי התור**: מי שתורו מקבל באנר "🎙️ תורך!" עם צליל ורטט, ולוחץ "אמרתי ✓" כדי להעביר הלאה. המנחה יכול לדלג בכל רגע.
5. **מצביעים** מי המתחזה. אם תפסתם אותו — החבורה מנצחת. אם לא — המתחזה מנצח.
6. סבב נוסף, וניקוד מצטבר בין הסבבים.

## פריסה אונליין (לינק ציבורי לשחק ממש)

הדרך המהירה ביותר לקבל לינק שכולם יכולים להיכנס אליו מהאינטרנט היא **Render** (יש שכבת חינם שתומכת ב-WebSocket):

[![Deploy to Render](https://render.com/images/deploy-to-render-button.svg)](https://render.com/deploy?repo=https://github.com/AzMaVeZe/MitHaze)

1. לוחצים על הכפתור, מתחברים ל-GitHub ומאשרים את המאגר.
2. Render מזהה את `render.yaml`, מריץ `npm install` ומפעיל `npm start`.
3. תוך דקה-שתיים מקבלים כתובת כמו `https://mithaze.onrender.com` — זה הלינק לשחק!

> בשכבת החינם השרת "נרדם" אחרי חוסר פעילות; הכניסה הראשונה אחרי הפסקה עשויה לקחת כ-30 שניות עד שהוא מתעורר.

אפשר גם לפרוס ב-**Railway** / **Glitch** / **Replit** באותו אופן (ייבוא מ-GitHub, פקודת הרצה `npm start`).

## הפעלה מקומית

```bash
npm install
npm start
# פתחו http://localhost:3000
```

השחקנים שמתחברים מאותה רשת (Wi‑Fi) צריכים להיכנס לכתובת ה-IP של המחשב שמריץ את השרת
(למשל `http://192.168.1.20:3000`) ולהקליד את הקוד, או פשוט לסרוק את ה-QR שמופיע במסך המנחה.

משתנה סביבה `PORT` קובע את הפורט (ברירת מחדל 3000).

## הגדרות משחק

- **קטגוריה** — חיות, אוכל, מקומות, מקצועות, ספורט, חפצים, סרטים… או "הכול מעורבב".
- **מספר מתחזים** — 1 עד 3.
- **המתחזה רואה את הקטגוריה** — מקל/מקשה על המתחזה.
- **המנחה משתתף כשחקן 📱** — סמנו את התיבה בזמן היצירה כדי לשחק גם מהטלפון של המנחה: אותו מכשיר שולט במשחק (התחלה/הצבעה/חשיפה) וגם מקבל תפקיד סודי ומצביע כמו כולם. בלי לסמן — מסך המנחה משמש כמסך משותף (למשל טלוויזיה/מחשב) בלבד.

## שיתוף והצטרפות

במסך הלובי של המנחה יש שלוש דרכים להזמין:
- **QR** — סורקים ומצטרפים ישירות.
- **📲 שתף בוואטסאפ** — פותח וואטסאפ עם הודעת הזמנה מוכנה (לינק + קוד) לשליחה לקבוצה. כל אחד לוחץ על הלינק ומצטרף מהטלפון שלו — התפקידים נשארים סודיים.
- **🔗 העתק לינק** — משתף/מעתיק את הלינק (משתמש ב-Web Share אם זמין).

## מבנה הפרויקט

```
server.js         שרת Express + Socket.IO, נקודות קצה ל-QR ולקטגוריות
src/rooms.js      ניהול חדרים ולוגיקת המשחק (חלוקת תפקידים, הצבעה, ניקוד)
data/words.js     קטגוריות ומילים בעברית — קל להוסיף/לערוך
public/           לקוח (HTML/CSS/JS), ממשק RTL מותאם למובייל
```

## טכנולוגיה

- **Node.js + Express** להגשת הלקוח ולנקודות קצה.
- **Socket.IO** לסנכרון בזמן אמת (הצטרפות, חלוקת תפקידים פרטית, הצבעות).
- **qrcode** ליצירת קוד ה-QR להצטרפות.
- לקוח וניל (ללא build), עם התחברות מחדש אוטומטית (localStorage) אם הטלפון מתנתק.

התפקידים (מי המתחזה / מהי המילה) נשלחים **בפרטיות** לכל שחקן ולעולם לא משודרים למסך המשותף.
